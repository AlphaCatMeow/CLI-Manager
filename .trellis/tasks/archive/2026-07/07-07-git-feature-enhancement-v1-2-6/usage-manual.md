# V1.2.6 Git 分支工作流使用手册

## 入口

1. 打开一个绑定本地 Git 项目的终端。
2. 点击终端右侧工具栏的 Git 变更入口。
3. 在 Git 变更面板底部找到当前分支行，点击分支名或分支图标打开分支菜单。

## Fetch 远端信息

1. 打开分支菜单。
2. 点击 `Fetch 远端`。
3. 应用执行 `git fetch --prune`，只刷新远端分支和提交信息，不执行 merge/rebase，不修改工作区文件。

## 切换本地分支

1. 打开分支菜单。
2. 在搜索框输入分支名片段缩小列表，或直接在 `本地分支` 区域点击目标分支。
3. 应用执行普通 `git checkout <branch>`。
4. 切换成功后会刷新变更列表、当前分支、ahead/behind、仓库列表和分支列表。

## 搜索分支

1. 打开分支菜单。
2. 在搜索框输入本地分支名、远程分支名或 upstream 片段。
3. 本地分支和远程分支会实时过滤。
4. 如果分支很多，菜单只渲染每组前 80 条，并提示还有多少结果；继续输入可缩小范围。

## Checkout 远程分支

1. 先点击 `Fetch 远端`，确保远程分支列表最新。
2. 在 `远程分支` 区域点击目标分支，例如 `origin/feature/demo`。
3. 应用执行 `git checkout --track origin/feature/demo`，创建本地跟踪分支并切换过去。
4. 如果本地已有同名分支，Git 会返回错误；此阶段不自动覆盖或重命名。

## 新建并切换分支

1. 打开分支菜单。
2. 在搜索框输入新分支名，例如 `feature/git-branch-menu`。
3. 点击 `新建并切换：<branch>` 按钮。
4. 应用执行 `git checkout -b <branch>`，从当前 HEAD 创建新分支并切换过去。

## 安全边界

- 不支持强制 checkout。
- 不自动 stash、shelf 或迁移本地未提交改动。
- 当 checkout 会覆盖本地改动时，应用显示“本地改动会被覆盖，请先提交或暂存”。
- 分支名会在后端通过基础规则和 `git check-ref-format --branch` 校验。
- 所有操作都针对 Git 面板当前选中的仓库；如果项目有子仓库，请先在仓库下拉中选中目标仓库。
