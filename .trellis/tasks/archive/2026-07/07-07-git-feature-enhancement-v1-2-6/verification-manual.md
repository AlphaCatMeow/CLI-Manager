# V1.2.6 Git 分支工作流验证手册

## 自动检查

在仓库根目录执行：

```powershell
npx tsc --noEmit
```

在 `src-tauri` 目录执行：

```powershell
cargo check
```

可选：只跑本次新增的分支名校验测试：

```powershell
cargo test branch_names
```

## 手动验证准备

1. 准备一个有 remote 的本地 Git 项目。
2. 至少准备两个本地分支，例如 `master` 和 `feature/local-switch-test`。
3. 至少准备一个尚未 checkout 到本地的远程分支，例如 `origin/feature/remote-checkout-test`。
4. 打开 CLI-Manager，进入该项目终端，打开 Git 变更面板。

## 场景 1：分支列表与当前分支

1. 点击 Git 变更面板底部当前分支。
2. 确认菜单显示 `本地分支` 和 `远程分支` 两组。
3. 当前本地分支应带选中标记。
4. 有 upstream 的本地分支应显示对应远端跟踪分支。

预期：菜单无明显遮挡；分支名过长时截断，不撑破面板。

## 场景 1.1：分支搜索与输入性能

1. 打开分支菜单。
2. 在搜索框连续输入分支名片段。
3. 确认本地分支、远程分支会实时过滤。
4. 搜索无结果时显示空状态。
5. 分支数量超过单组渲染上限时，应显示“还有 N 个结果”提示。

预期：输入搜索时主变更文件树不应跟随明显卡顿；搜索框不再被当作默认新建分支入口。

## 场景 2：切换本地分支

1. 在 `本地分支` 中点击非当前分支。
2. 等待 toast 显示切换成功。
3. 确认底部当前分支名变为目标分支。
4. 确认变更列表、ahead/behind 状态刷新。

预期：没有强制覆盖工作区；失败时显示可读错误。

## 场景 3：Checkout 远程分支

1. 点击 `Fetch 远端`。
2. 在 `远程分支` 中点击尚未 checkout 的远程分支。
3. 等待 toast 显示切换成功。
4. 通过终端执行 `git branch --show-current`，确认已创建并切换到本地跟踪分支。
5. 再次打开分支菜单，确认新分支出现在本地分支列表。

预期：使用 Git 自身的 `--track` 语义建立 upstream。

## 场景 4：新建并切换分支

1. 在分支菜单搜索框输入一个合法新分支名。
2. 点击 `新建并切换：<branch>` 按钮。
3. 确认 toast 显示新建成功。
4. 通过终端执行 `git branch --show-current`，确认已切换到新分支。

预期：空分支名无法提交；非法分支名显示“分支名无效”。

## 场景 5：Fetch 不修改工作区

1. 在当前分支制造一个未提交修改。
2. 记录 `git status --short` 输出。
3. 在分支菜单点击 `Fetch 远端`。
4. 再次执行 `git status --short`。

预期：本地未提交改动仍存在；Fetch 只更新远端信息，不合并、不改文件。

## 场景 6：Checkout 冲突保护

1. 在当前分支修改一个文件但不提交。
2. 选择一个会覆盖该文件内容的目标分支。
3. 在分支菜单点击该目标分支。

预期：checkout 失败，提示本地改动会被覆盖；当前分支不变，工作区文件不被覆盖。

## 场景 7：中英文文案

1. 设置 -> 通用 -> 界面语言切到中文。
2. 打开 Git 分支菜单，检查按钮、分组、toast、错误提示为中文。
3. 切到英文。
4. 重复检查分支菜单和 toast。

预期：新增可见文案都走 i18n；英文界面不会出现硬编码中文。

## 场景 8：子仓库

1. 打开包含子仓库的项目。
2. 在 Git 面板顶部仓库下拉中切换到子仓库。
3. 打开分支菜单并执行分支列表、Fetch 或切换。

预期：操作只作用于当前选中的仓库，不误操作项目根仓库。
